1. **引言**

**目的**

1.**确定美术资源制作的标准，为美术制作提供依据。**

2.**详细描述美术制作流程的各个环节。**

**文档约定**

**预期读者和阅读建议**

**预期读者**

角色及动画设计制作人员、场景设计及制作人员、特效制作人员

**阅读建议**

相关人员应尽量仔细的阅读整个文档。

**文档的范围**

本文档涉及整个项目的所有美术制作的规范

Unity3D：一款强大的跨平台3D游戏引擎。

 

2.**工作环境及配置**

**Unity3D**

本项目使用Unity3D引擎进行制作。

版本：2018.3.5f1（请统一使用此版本）

**建模工具**

支持的建模工具：3DMax、Maya、Blender等

关于支持的软件及引擎导入说明请参考：

http://docs.unity3d.com/Manual/HOWTO-importObject.html

**坐标系统**

Up axis : Y

**单位与比例**

请首先按照下列参数设置建模环境

3DMax版本：2012版本

 Maya版本：2012版本

3DMAX:

1 Units= 1 Centimeters,设置使用 Generic Units 

 

Maya:

Linear = centimeter

 

 

在实际建模中，建模软件内的1个单位代表1厘米。如一个角色身高1.65米，在建模软件内设置为165。

**环境检查**

在MAX中制作一个尺寸为 100 x 100 x 100 的正方体，导入Unity，放入场景中，然后在创建一个 Cube，两者的大小如果一致，则环境正确。

4.**命名规则**

物件命名实例

**Demon_O_shan02**     level名+类型名+对象名

- 类型名字：建筑 Build（B） 植物Plant（P）杂物 Object（O）
- 所有对象命名必须使用有意义的英文表示，多个同类对象可追加数字后缀

如草地：grass01、grass02......

- 命名严格区分大小写贴图根据类型增加后缀：

漫反射贴图(Diffuse)：_D

高光贴图(Specular Map)：_S

法线贴图(Normal Map)：_N

自发光（Self-Illumin Map）：_I

贴图和模型应位于相同目录。

每个独立的角色、场景应放置到独立的目录。

5. **贴图**

**文件格式**

原始格式：PSD格式

输出格式：TGA、PNG

**场景贴图**

1/2倍角色高度物体 64以下（可以根据需要使用更小值或者合并贴图使用）

1倍角色高度128以下 （一般情况下均使用此尺寸，可以根据需要使用更小值。）

2倍角色高度 256以下（除非细节要求较高，否则尽量不使用。）

3倍以及以上角色高度 512-1024（除非极端情况，否则不应使用。）

合并纹理：合并纹理可突破上述限制，最大可使用1024 x 1024。

6. **材质**

**基本要求**

- **材质球命名与物体名称一致，材质球的父子层级的命名必须一致**
- **同种贴图必须使用一个材质球**
- **除需要用双面材质表现的物体之外，其他物体不能使用双面材质**

7. **场景制作**

**基本要求**

- **单个场景所有物件的最大三角面应限制在20000个三角形内。**
- **单个物件的顶点数尽量控制在500个顶点内，UV空间在（0，1）区间。**
- **尽可能重复利用现有的模型，通过旋转比例等产生变化。**
-  **单个物件只可包含一个网格。**
- **在** **Unity** **引擎内创建** **LightMap和lightprobe** **代替动态光照。**
- **整个场景透明贴图控制在2张。**

**文件结构**

- **所有场景的unity文件存放在Level下面，对应场景的名字在Level下面建立一个相同的文件夹，这个文件夹里面放里面放fbx，同时在这个文件夹下面建立Materials和Textures文件夹里面分别放材质和贴图。**

 

![img](https://i.postimg.cc/7Pj11XpY/image_8.png)

 

**文件命名**

- **所有文件请采用跟物体对应有实际意义的英文命名**

MAX里面的物体命名，材质球名字，贴图名字，fbx名字保持统一，unity里面的物体名字跟max里面保持统一

- **Hierarchy面板清晰整洁如下图，放入关卡中的物体需要放到对应的Group**

 

![img](https://i.postimg.cc/tR8ddDL1/image_9.png)

 

**Lightmap**

- **使用single lightmap方式烘培**
-  **普通战斗场景256\*256大小**
- **要保证填满整张贴图**

![img](https://i.postimg.cc/MZg00thQ/image_10.png)

 

- **注意事项** 

如果碰到贴图UV空间还有一些空余的时候，微调高resolution值烘培会出第二张lightmap的情况，微调个别物体的lightmap scale，保证一个level只有一张lightmap。（如下图：左边有浪费需要调整，右边利用率更高）

 

- **地面模型与lightmap**

除了玩家行走区域外的模型顶点尽可能的优化。只保留对结构起作用的部分。镜头以外彻底看不到的区域删除掉，否则会占用lightmapUV空间，导致降低lightmap最终在主要游戏区域精度降低。

- **每个物件需要设置物体的lightmap scale,硬边缘的物体还需要更改Max Angle，默认设置为65**

**关于模型精度控制**

- **优化掉不必要的小的结构造型，在一些结构轮廓变化不是很大的模型上没必要给太多段数。**

比如这个蘑菇柄的部分段数太多了，伞状结构的也完全没必要给太多面。把面用在对轮廓结构起作用的位置即可。



- **模型精度占画面比例的统一性**

**场景中的所有物体模型的顶点，布线密度应该是比较均匀的，不要有某些小的东西精度却很高这种情况。**

- **关卡中同一高度的模型要注意精度的统一，不要出现明显的贴图精度差**
-  **如下图出现的这种**

 ![img](https://i.postimg.cc/nc743BJC/image_12.jpg)

**水的shader**

- **请不要使用unity标准资源里面的水面模型，请单独建模匹配地形结构的水面。不要有大面积的看不见的区域插入地面。**

**shader请使用water** **（simple）**

**场景和摄像机的位置方向**

- **场景主路中心线在Z轴负方向，场景左侧边缘与主路中心线交叉点坐标在X:0,Y:0,Z:0的位置。**
- **主路的宽度800，装饰性的物件摆在两侧，整体保持800左右的范围。**
- **摄像机请参照以下设置**



**战斗区域**

每个场景应设定一个或多个“矩形”的“平坦”战斗区域。

战斗区域应保证绝对平坦且无任何障碍。

竞技场为单次战斗，战斗区域为400 x 2000。

关卡战斗每个场景包含3次战斗，每次战斗区域为400 x 2000。

两个战斗区域之间为行走区域，行走区域默认长度为0，最大范围在（0-800）之间，如需设计桥梁、台阶等特殊通路时，可适当增加行走区域，否则一律为0。（注：战斗区域两个各有200长度的预留安全区域，如果有小型台阶，长度不超过400的，可不必增加行走区域）。

如下图图所示，这是一个关卡战斗场景，包含3个战斗区域，战斗区域间的蓝色区域为行走区域。

 

整个场景应完整包含战斗区域，场景两侧可略作空间预留。

8. **角色制作**

**基本要求**

- **模型文件按角色代码进行命名**
- **材质球名称与漫反射贴图名称应保持一致
- **角色三角面数建议在1500-2000之间，最大三角面数限制在3000**
- **角色骨骼数建议限制在30，极限值不可超过50**
- **每个角色应只尽量包含一个网格对象。**

**贴图**

每个角色贴图建议采用一张贴图，特殊材质请财考【4.4角色贴图】部分。

默认情况下，我们建议角色使用高光贴图，并将高光贴图放入颜色贴图的Alpha通道中，以节省贴图。

 

贴图请放在模型所在目录下的子目录Textures。

**命名规则**

怪物命名前缀 M 比如M701_LOD1

角色命名前缀 C比如C411_LOD1；C1195_UI

**骨骼与蒙皮**

注意：请在蒙皮之前，务必先导入模型到Unity3D中进行检查，确定尺寸、法线（请放入方向光检查模型的明暗与光照方向是否一致）、材质效果等均正常，然后再开始进行蒙皮。

支持 Bip 骨骼系统，支持附加 Bone，不支持虚拟体。

支持 Skin 蒙皮(建议)

附加骨骼请使用有意义的命名，参考列表：

| ----------- | -------------- | ---- |
| WeaponLeft  | 左手武器绑定点 |      |
| WeaponRight | 右手武器绑定点 |      |
|             |                |      |

9. **特效制作**

**基本要求**

-  **严格控制粒子数量与面数**
- **特效材质请尽量使用** **Mobile/Particles下的Shader，特殊情况除外**
- **所有特效贴图统一放置在** **Assets/Art/FX/Textures**
- **所有特效材质统一放置在** **Assets/Art/FX/Materials**
- **所有特效模型统一放置在** **Assets/Art/FX/Modes**

**命名要求**

- **公用技能特效：[前缀]_[名称]**
- **角色技能类特效：[前缀]_[角色代码]_[动作后缀]**
- **角色非技能特效：[前缀]_[角色代码]_[名称]**
- **场景特效：[前缀]_[场景名]_[名词]**
- **UI特效：[前缀]_[名称]**

说明:

- **[前缀]：参考特效类型表中的命名前缀**
- **[角色代码]：通常为1位字母和多位数字，如C001, M001**
- **[动作后缀]：atk、atk2、atk3、ult**
- **[名称]：代表特效的名称、类型、或行为的英文命名，如：fire、light、sleep、open等**
- **角色技能特效前缀必须为fx_skill,非技能特效不能使用fx_skill前缀，每个角色身上同名技能特效只能存在一个。**

**粒子数与Particle System**

-  **一个粒子系统大概是24kb，所以请尽量用比较少的粒子系统来创建出合适的效果。用的粒子系统越多最终prefab文件也越大，也会导致总的资源变大。**


**Prefab里面的粒子系统无用的或者临时的部分提交前删除掉，否则会增加文件大小**

**一个特效最多不要超过五个不同材质的粒子系统，能少尽量少用。**

**同样也尽量减少使用材质。材质，粒子系统都会增加drawcall数量，将直接影响性能。**

 

 

- **粒子数量：粒子数量的多少将直接影响游戏运行的帧率，尽量保证最节省的使用粒子，不要为了所谓的完美效果大量的使用粒子。一个普通的特效正常情况下几十个粒子足够。**

**每个粒子系统最多控制在15个左右的粒子数，保持合理利用，不要每个都用满15个粒子，极个别特殊的效果可以适当微调。**

 

下图中左边的效果和右边的效果基本相似，但是左边的只用了2个粒子，右边的却用了5个粒子。如果这个特效作为某个特效的一个元素出现了很多，这将非常浪费粒子。

 

 

**Particle System设置**

-  **同一个特效最顶命名层必须是一个Particle System，不能是一个空物体。**


 

**并且把play on awake** **关掉**

-  **大多数情况下不用粒子碰撞，除非特殊效果的特效，并且是比较重要的特效。**
- **关闭每个发射器的接受和投射阴影渲染选项**
- **角色身上的特效请选择world模式，这样角色在移动的时候特效会产生自然的拖尾效果。当然一些特殊效果的特效是不需要拖尾效果的，这个就自己控制了。**

角色技能特效无特殊情况必须使用Local模式。

 

 

**贴图规范**

- **贴图大小限制在128x128**
- **请尽量使用灰度贴图，通过粒子系统控制颜色**
- **强化资源复用，严格限制同类效果贴图的数量**

**错误示例：**

1. **同样的效果，制作了不同的贴图。**
2. **近似的效果有太多的贴图，可按详细程度进行适当合并。**

 

- **特效贴图尺寸最大128\*128**

除非极个别情况，需要表现的特效贴图细节比较多，而且占游戏画面比例比较大，可以用256*256

如果遇到长条形贴图，请尽量减少长度。（可以按照正常比例去绘制贴图，然后最终通过缩放把尺寸控制在一个适中的范围，尽量不要出现过长的贴图）如以下步骤

最多长宽比不要超过四倍。

1. 最初的贴图
2. 把长度等比例缩减一半
3. 最终在特效中通过粒子重新拉成正常的比例

- **保存tga格式的图片，背景为纯黑色用作通道，不要制作透明通道否则文件压缩后会大一倍，除非个别情况需要制作透明通道。**
- **不要为同一张贴图去制作不同颜色版本的贴图，这样非常浪费资源。**
- **使用一张灰度图作为这张贴图的原形，颜色通过unity里面的特效系统赋予颜色，这样就完全不需要为不同的贴图效果分别制作贴图和材质了。**
- **贴图不要出现大面积无用的区域，非常浪费贴图，而且会增加渲染压力。**
- **像下面这种重新切割后贴图大小将下降很多，而且不会影响品质。**

另外在引擎中，如果按照上面的那张贴图做为特效贴图的话，那么粒子势必也是一个方形的，这会导致大面积的透明区域在场景中参与渲染，会严重影响帧率。

- **在制作特效的时候尽量主动思考用什么方式可以达到最佳的利用效率。在做每个特效的时候都要考虑粒子系统，粒子数量，贴图张数，材质数量。**

  **比如下面这个例子，特效元素类型基本一致，只是发射的粒子贴图不一样，这样分开四个粒子系统去做实际上太浪费了。**    

 

 

 

 

应该把四张贴图合并在一起，然后用贴图动画的方式制作

这样只需要一个发射器，一张贴图，一个材质。并且通过贴图动画的具体参数调节可以控制更好的随机感。最终实现基本一样的效果但是却大大降低了特效的复杂度。

 

- **如果需要引用角色或者场景的模型贴图，需要减面和缩小贴图尺寸来适配特效**

**下面这张角色贴图，特效只是引用了很小的一块区域，却把整张贴图完整保留，这是非常浪费的一种情况**

 

**采用正确的思路虑去构建特效**

- **下面这个闪电是特效中的一部分，持续时间半秒钟左右，这其实是一个比较普通的球形闪电，但是却用了四个粒子系统来表现，而且分别用了四张不同的闪电贴图。**

（如果这个闪电持续时间比较长那么就需要用贴图动画的方式来增加丰富度）

- **下面这个特效占用了10个draw call** **太浪费了，光烟就四个粒子系统。像是这种特效就需要做简化处理，来降低drawcall数量。**
- **在制作特效的时候可以打开Game窗口下的stats，你会看到当前这个特效的一些详细的统计数据，包括draw call** **数量，顶点数量等等。**
-  **做BUFF类特效的时候，需要勾上全部粒子系统的Prewarm。以实现特效一出生，就显示最完善的姿态。在遇到需要永久持续显示的粒子的时候，可以如此设置，这样就能不间断的显示粒子，而且不会出现突然闪烁的BUG。**
- **遇到需要显示很大的粒子片的时候，要把Max Particl Size设置成** **≥ 1，如下图所示。**
- **不要用Unity里面自带的多边形MESH，尽量可以MAX创建更低面数的MESH来使用，而且可以简化删除一些用不到的面。**
-  **手游特效的设计，尽量以大块面，少粒子数来制作。制作特效前，按照策划案，找到最明显的造型点，夸大它，让玩家一眼就看出它是什么特效。如下图中，策划原意是设计一个瞌睡的特效。右边的感觉ZZZ不够明显，而且竖线的粒子又短又多又不明显。所以，如图左边所示那样，放大了ZZZ标志，加长放大竖线条，不仅效果明显了，粒子数也是减少了很多。**
- **合理利用MESH，来制作特效。当碰到粒子无法制作的效果的时候，或者需要很多很多粒子才能表现效果的时候，可以去从MESH角度来制作。如下图中的海浪为示，如果用粒子表示中间的大海浪的效果，会需要很多很多粒子。但是如果用MAX制作的MESH，然后再滚动UV的方法来表示海浪，就会省很多面。仅需要在海浪的首尾，加点特效的小粒子即可。**
-  **简化粒子系统这个很重要，很重要。因为在手游中，粒子特效占很大的消耗。所以，尽量用少的粒子，来表现最明显的效果。下图中，同样的表达光盾旋转的，左边就比右边省了很多很多的面数。效果上没有变化。**
- **如下图所示，下方的超声波，为了让他在最大的时候结束，而把之后的MESH的Alpha设置为0，这样做虽然看不到，但GPU依然在计算它的顶点和面，占用性能，所以这样做是错误的。按照上面那样做，用生命周期来控制显示，才是最省资源的。**
